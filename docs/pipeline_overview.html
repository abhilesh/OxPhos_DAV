OXPHOS DAV Pipeline Overview

Goals, data sources, analysis stages, and current results

Author

OxPhos DAV project

Published

May 15, 2026

1 Purpose

This document is a living overview of the OXPHOS disease-associated variant pipeline. It records the scientific goals, major data sources, stage contracts, and current headline results. Detailed implementation notes remain in notes/ and stage-specific scripts under src/.

2 Scientific Goals

Goal Operational definition Main output
Build an auditable OXPHOS DAV inventory Retain disease-associated variant records from MITOMAP and ClinVar with explicit source, eligibility, and exclusion metadata variants_master_curated.parquet
Classify compensated DAVs AA-level cDAV: a non-human species carries the human disease amino acid. NT-level cDAV: a non-human species carries the exact human disease codon variants_master_classified.parquet
Preserve unresolved and ineligible cases Use filter-late metadata rather than silent row dropping classified master, QC files, mismatch logs
Map classified variants to structure Anchor variants in primary and validation OXPHOS structures with chain, residue, and contact provenance dar_structure_map.csv, dar_contacts_cbcb8A.csv
Identify candidate compensating partners Test whether structural neighbours of cDAV positions show co-evolutionary enrichment all_tested_pairs.csv, compensatory_partners.csv
Prepare downstream mechanistic analyses Support phylogenetic timing, DCA, and mutagenesis/FoldX analyses src/phylo/, src/mutagenesis/, HPC scripts

6.4 Gene-symbol Boundary

The frozen HGNC source uses COXFA4 as the approved analytical symbol and NDUFA4 as a previous/resource symbol. Structural resources still commonly use NDUFA4. Current Stage 3 outputs therefore separate:

Field Meaning Example
interpreted_gene, dar_locus, contact_gene HGNC/classification analytical key COXFA4
structure_gene, dar_structure_gene, contact_structure_gene PDB/UniProt structural-resource key NDUFA4

8 Known Caveats

Caveat Current handling
MyVariant/dbNSFP/gnomAD completeness Marked legacy-unchecked; not used as a primary inference filter
MITOMAP programmatic access Local file validated; remote HTTP 403 documented
mtDNA ClinVar rows Retained for traceability, linked to MITOMAP where possible, ineligible for nuclear comparative branch
Residual unresolved nucDNA rows Retained as unresolved, never forced into uDAV
Structural mappability Structural conclusions conditioned on mapping eligibility/support class
Duplicate protomers Gene-level mapping is usable; protomer-specific claims require caution
Local Pagel runtime Full phylogenetic testing is HPC-bound; local runs may be incomplete or partial
